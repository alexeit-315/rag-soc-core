## 1. Функциональность (Functionality)

### 1.1 Основные функции

| ID   | Требование                                                                                                                                                     | Приоритет | Комментарий                                     |
|------|----------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------|-------------------------------------------------|

| F-02 | Система должна поддерживать интерактивный режим вопрос-ответ (чат)                                                                                             | MUST | Для повседневной работы                         |

| F-04 | Система должна предоставлять ответы в свободной форме со ссылками на статьи-источники                                                                          | MUST | Основа для верификации                          |

| ID   | Требование                                                                                                                                                                                             | Приоритет | Комментарий                                                                       |
|------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------|-----------------------------------------------------------------------------------|
| F-40 | Система должна поддерживать настраиваемые стратегии чанкинга (size, overlap, semantic chunking)                                                                                                        | MUST |                                                                                   |
| F-41 | Система должна поддерживать гибридный поиск (semantic + keyword/BM25)                                                                                                                                  | MUST | Для BM25 используется Elasticsearch                                              |
| F-42 | Система должна поддерживать reranking retrieved контекста                                                                                                                                              | MUST |                                                                                   |
| F-43 | Система должна предоставлять explainability на основе retrieved контекста, а не внутреннего reasoning модели                                                                                           | MUST |                                                                                   |
| F-44 | Система должна отклонять ответы при недостаточном контексте (no-answer fallback)                                                                                                                       | MUST |                                                                                   |
| F-45 | Система должна автоматически выбирать релевантные индексы на основе метаданных запроса                                                                                                                 | MUST |                                                                                   |
| F-46 | Система должна сохранять при парсинге html filename, dc identifier, document type, title, hierarchy (breadcrumbs), product series, firmware versions, features set, validation flag в метаданных чанка | MUST | Расширение метаданных при парсинге                                                |
| F-47 | Система должна сохранять при парсинге ссылки на связанные документы в метаданных чанка                                                                                                                  | MUST | Расширение метаданных при парсинге                                                |
| F-48 | Система должна поддерживать section-aware retrieval с возможностью фильтрации по "тот же раздел"                                                                                                       | SHOULD | Требует поддержки в векторной БД (фильтрация по метаданным)                       |
| F-49 | Система должна поддерживать query preprocessing: нормализацию и расширение на основе словаря концептов (синонимы, аббревиатуры)                                                                        | SHOULD | Перефразирование запросов без добавления новых фактов                             |
| F-50 | Система должна поддерживать динамический отбор чанков после реранкинга (относительный порог, а не жесткий)                                                                                             | MUST |                                                                                   |
| F-51 | Система должна структурировать ответ: краткий ответ (1-3 предложения) + развернутый ответ с цитированием по предложениям + источники                                                                   | MUST | Структура ответа                                                                  |
| F-52 | Система должна предоставлять trace-режим для отладки: query → normalized → retrieved → reranked → rejected(reason)                                                                                     | SHOULD | Реализуется через флаг `?trace=true` в API или отдельный эндпоинт                 |
| F-53 | Система должна оптимизировать обработку SOC за счет переиспользования retrieval результатов между вопросами                                                                                            | SHOULD | Оптимизация в batch-режиме                                                        |
| F-54 | Система должна поддерживать fallback стратегии (другая модель / упрощенный ответ / retrieval-only)                                                                                                     | SHOULD | Explicit model fallback                                                           |
| F-55 | Система должна поддерживать связь терминов (синонимы, parent-child, aliases)                                                                                                                           | COULD |                                                                                   |

## Приложение: Метрики качества LLM (для справки)

| Метрика            | Назначение                                  |
|--------------------|---------------------------------------------|
| latency            | Задержка ответа                             |
| recall@k           | Полнота найденных релевантных чанков        |
| precision@k        | Точность найденных чанков                   |
| nDCG               | Качество ранжирования                       |
| faithfulness       | Следует ли ответ из контекста               |
| answer correctness | Правильность ответа                         |
| context precision  | Релевантность контекста запросу             |

*Примечание: метрики автоматически собираются в тестовом контуре и используются для сравнения моделей и гиперпараметров.*
