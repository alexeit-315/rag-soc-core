# Инструкция по развертыванию среды контейнеризации

Ниже приведена инструкция по установке инфраструктуры для Docker Compose и Helm на Windows с использованием **Git Bash** (эмулятор bash от Git для Windows).

Особенности для Git Bash:
- Используются **Unix-подобные пути** (`/c/Program Files/...` вместо `C:\Program Files\...`)
- Команды **curl**, **wget** и **unzip** часто уже доступны "из коробки"
- Синтаксис команд соответствует Linux/macOS, что удобнее для работы с Helm и kubectl

---
Ошибка, которую вы получили, означает, что ваш компьютер не готов к работе с WSL 2 из-за двух причин:

1. **Виртуализация (VT-x/AMD-V) отключена в BIOS/UEFI**
2. **Отсутствует необходимый компонент Windows "Платформа виртуальной машины"**

Ниже приведена пошаговая инструкция по исправлению этой ситуации.

---

### Шаг 1: Включение виртуализации в BIOS/UEFI

Вам нужно войти в BIOS (базовую систему ввода-вывода) при загрузке компьютера и включить технологию виртуализации.

**Процесс:**
1. **Перезагрузите компьютер**.
2. Войдите в BIOS. Обычно это:
3. Найдите настройки виртуализации. Название зависит от производителя процессора:
    - **Intel**: *Intel Virtualization Technology* (VT-x), *VT-d*
    - **AMD**: *SVM Mode* (Secure Virtual Machine), *AMD-V*

   *Часто эти настройки находятся в разделах: Advanced, CPU Configuration, System Configuration.*

4. Установите значение **Enabled** (Включено) для этих опций.
5. Сохраните изменения и выйдите из BIOS. Компьютер перезагрузится.

---

### Шаг 2: Установка WSL 2 (рекомендуется)

WSL 2 обеспечивает лучшую производительность Docker на Windows.

1. Откройте **PowerShell или cmd от имени администратора** (Git Bash не подойдет, так как требует прав администратора):
   ```powershell
   wsl --install
   ```

2. После установки **перезагрузите компьютер**.

---

### Шаг 3: Установка Docker Desktop

1. Скачайте Docker Desktop с официального сайта (https://www.docker.com/products/docker-desktop/).
2. Запустите установщик и следуйте инструкциям.
3. **Важно:** Убедитесь, что опция **"Install required Windows components for WSL 2"** отмечена.
4. После установки запустите Docker Desktop.

---

### Шаг 4: Настройка и запуск Kubernetes

1. В Docker Desktop откройте **Settings** (значок шестеренки).
2. Перейдите в раздел **Kubernetes**.
3. Установите флажок **"Enable Kubernetes"**.
4. Нажмите **"Apply & Restart"**. Дождитесь, пока индикатор внизу слева станет зеленым.

---

### Шаг 5: Установка Helm

Helm устанавливается отдельно. Используйте один из способов ниже.

#### Ручная установка

```bash
# Скачать архив с последней версией (проверьте актуальную версию на helm.sh)
curl -LO https://get.helm.sh/helm-v3.15.0-windows-amd64.zip

# Распаковать (если unzip не установлен, используйте команду ниже)
unzip helm-v3.15.0-windows-amd64.zip

# Переместить helm.exe в директорию, доступную в PATH
mv windows-amd64/helm.exe /c/Program\ Files/Docker/Docker/resources/bin/
```

### Проверка работоспособности всего стека

Выполните последовательно в Git Bash:

```bash
# 1. Убедиться, что Docker работает
docker --version
docker ps

# 2. Убедиться, что Kubernetes работает
kubectl cluster-info
kubectl get nodes

# 3. Убедиться, что Helm работает
helm version

# 4. Проверить Docker Compose
docker compose version
```
